Ch 10

  • Types of errors
    • System error - something the program is interacting with, device, or other system software generates an error
    • Programmer error - issue is with the directly in the programmers code
    • User error - program is unable to correctly handle the user interaction - probably because the user did something unexpected
  • Exception is an error with a return value
  • Stack trace - sequence of function calls (stack) that informs about exactly where the issue occured
  • Warnings - "A warning can occur if there’s an error in the code that isn't enough to cause the program to crash" Its an informational statement that the programmer may want to be aware of, but may have also done intentionally.
  • Strict mode
    • 'use strict'; on first line of a javascript file (will enable strict mode globally
    • better per function:
      function func(){
      'use strict';
      ...}
    • best - self invoking function:
      (function() { 'use strict'; // All your code would go inside this function }());
    • modules are strict by default
  • Linting - helps highlight issues and bad practices
    • JS Lint
    • JS Hint
    • ES Lint
  • Feature detection - browser support for features is behind the specifications, and users may not use the latest browsers anyway
    • Check if the appropriate object exists and fail gracefully
    • Modernizr is a library that offers an easy way to implement feature detection
    • Can I Use?
    • do not use browser sniffing (using user agent string)
  • Debugging
    • Alert box
    • Console
    • debugger keyword - This will create a breakpoint in your code that will pause the execution of the code
  • Error objects - new Error("Error Message");
    • EvalError is not used in the current ECMAScript specification and only retained for backwards compatibility. It was used to identify errors when using the global eval() function.
    • RangeError is thrown when a number is outside an allowable range of values.
    • ReferenceError is thrown when a reference is made to an item that doesn’t exist. For example, calling a function that hasn't been defined.
    • SyntaxError is thrown when there’s an error in the code’s syntax.
    • TypeError is thrown when there’s an error in the type of value used; for example, a string is used when a number is expected.
    • URIError is thrown when there’s a problem encoding or decoding the URI.
    • InternalError is a non-standard error that is thrown when an error occurs in the JavaScript engine.
  • Throwing Exceptions
    try { ...code throwing exception...} catch(error) {code dealing with exception } finally { code always executed }
  • Test-Driven Development
    1. Write tests (that initially fail)
    2. Write code to pass the tests
    3. Refactor the code
    4. Test refactored code
    5. Write more tests for new features
  • Testing Frameworks
    • Jest is a TDD framework, created by Facebook
    • jest -c {} runs all files ending with .test.js in the folder
    • test("what is tested", ()=>{code and expectations})
    • expect(expression) creates expectation object
    • expect(expression).toBe(expectedResult) - expectation and a matcher
    • toBe, toEqual, etc. Jest's matcher methods
