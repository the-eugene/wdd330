HTML 5 & CSS3: Ch 8

  • CSS Transforms
    • transform the element without reflowing the document
    • default origin is element center, but can be set with transform-origin property
    • order of transform functions matters - the coordinate system gets transformed along with the element.
    • format is transform: func() func() ...;
    • translate(x,y) move element
    • scale(x,y) scales (ommiting the second argument causes uniform scaling
    • rotate(a) rotates the element
    • skew(x,y) skews the element on each axis separatey (x axis one arg.)
  • CSS Transitions
    • creates an animated transition between two states of an element - for example normal and :hover
    • transition-property: transform; defines which property will be animated
    • transition-duration: 0.2s; defines the time for the animation
    • transition-timing-function: ease-out; defines the curve for the speed of change
    • transition-delay: 50ms; create a delay before animation begins
    • multiple transitions can be applied using comma separated lists
    • transition: transform 0.2s ease-out 50ms, color 0.1s linear 50ms; shorthand property (this one sets 2 different transitions)
  • CSS Animations
    • creates keyframe states for an element and then transitions between them creating an animation
    • @keyframes SomeAnimation{} creates and names a set of keyframes
    • inside are rules - xx%{property: value;} etc.
    • animation-name: appearDisappear; Which keyframes to use for animation
    • animation-duration: 300ms; total time the animation will take
    • animation-timing-function: ease-in; timing function
    • animation-iteration-count: 1; number of times to repeat the animation
    • animation-direction: alternate; the direction in which to play animation (forward, backward, etc)
    • animation-delay: 5s; delay before playing the animation
    • animation-fill-mode: backwards; the state in which the element starts and ends the animation
    • animation-play-state: running; is the animation actively running.
    • Shorthand - animation: 300ms ease-in alternate 5s backwards appearDisappear; }

HTML 5 & CSS3: Ch 12

  • Canvas API
    • Make sure to set width and height using the HTML tag not CSS (CSS will not alter the coordinate system, only scale the result)
    • get canvas using JS then get context e.g. canvas.getContext("2d");
    • fill and stroke: context.strokeStyle = "red"; context.fillStyle = "blue";
    • context.fillRect(10, 10, 100, 100); context.strokeRect(10, 10, 100, 100); //draw a rectangle with fill and stroke
    • fillStyle can use colors, gradients, or patterns (patterns use Image objects)
    • Paths:
    • arc(x, y, radius, startAngle, endAngle, anticlockwise)
    • context.beginPath(); begins a new path
    • context.closePath(); closes the path
    • canvas5.toDataURL("image/png") Save the image!
    • drawImage(image, x, y);
    • context.getImageData(x, y, x2, y2); gets values from a section of the canvas (limited by cross-origin)
    • context.putImageData(imageData, 0, 0) writes data to canvas
    • context.font = "18px LeagueGothic, Tahoma, Geneva, sans-serif"; set font
    • context.fillText("Text", x, y); draw text
  • SVG
    • Built using XML
    • <svg xmlns="http://www.w3.org/2000/svg" viewbox="x y x2 y2"></svg>
    • <desc>Description</desc>
    • <rect x="10" y="10" width="100" height="100" fill="blue" stroke="red" stroke-width="3" /> rectangle
    • <path ... just use illustrator or inkscape to draw SVGs
    • Recommends Raphael - this is dated. D3.js seems to be the top framework these days.
  • Drag and Drop
    • set draggable to true
    • add event listener to dragstart, dragover, and drop
    • event.dataTransfer.setData("text/plain",data); use a DataTransferObject to transfer Data!
